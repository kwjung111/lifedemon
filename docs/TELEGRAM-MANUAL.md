# Life Daemon Telegram 사용법

## 가장 쉬운 사용법

명령어를 외울 필요가 없습니다. 일정, 할 일, 메모, 링크, 사진, 문서를 그냥 보내면 Life Inbox에 저장됩니다.

```text
2026-08-22 14:00 병원 예약
보험 갱신해야
메모 여권 사본은 서랍 안
https://example.com 나중에 확인
```

봇은 한 메시지로 저장 결과와 확인되지 않은 점을 알려줍니다. 고치거나 끝냈다면 그 확인 말풍선의 Telegram `답장` 기능을 사용합니다.

```text
23일 오후 2시로 바꿔
완료
취소해
```

## Inbox 일정과 정시 알림의 차이

날짜가 있는 내용을 그냥 보내면 Inbox에 저장되고 평일 아침 브리핑에 활용됩니다. 해당 시각에 Telegram 알림이 자동으로 오는 것은 아닙니다.

정시 알림이 필요하면 처음부터 `알려줘`라고 쓰거나, 저장 확인 메시지에 `알림도 등록해`라고 답장한 뒤 등록을 승인합니다.

```text
내일 오후 4시에 서류 발표 알려줘
```

## 나중에 다시 관리하기

`/inbox`는 한 번에 여덟 개를 보여줍니다. 목록 말풍선에 답장하면 예전 확인 메시지를 찾지 않아도 됩니다.

```text
2번 완료
1번을 23일 오후 2시로 바꿔
3번 취소
4번 보여줘
더 보여줘
```

링크는 목록에서 바로 열 수 있고, 저장한 Telegram 사진·문서는 `보여줘` 답장으로 다시 받을 수 있습니다. 지난 일정은 목록에서 표시하지만 다음 날 아침 행동 상위 목록에서는 제외합니다.

## 아침 브리핑

평일 09:00에 오늘 일정, Inbox 다음 행동, 주택, 채용 핵심을 한 메시지로 보냅니다. Inbox는 최대 세 개만 표시하며 해당 브리핑에 번호 또는 제목으로 답장해 완료·수정·취소할 수 있습니다.

주택이나 채용 추천을 더 보고 싶으면 평소 말투로 요청할 수 있습니다. 모든 자유문장은 전역 AI가 한 번만 해석하며, 목록 탐색·지원·피드백·Inbox·알림·운영 질문을 하나의 구조화된 결과로 구분합니다. 각 기능은 이 결과를 다시 해석하지 않고 검증 후 실행합니다. AI가 불확실하거나 응답하지 않으면 정규식으로 추측하지 않고 아무것도 변경하지 않습니다. 알림이 여러 개 쏟아지지 않도록 한 페이지씩 보여주며, 받은 목록에 자연스럽게 이어서 요청하면 다음 페이지로 연결됩니다.

```text
채용 다 보여줘
채용 전체 목록 보여줘
채용공고 싹 꺼내줘
주택 더 보여줘
```

## 공고가 안 보이는 이유 확인

회사명이나 공고 제목의 특징적인 부분을 넣어 평소처럼 물으면 됩니다.

```text
A회사 공고 왜 안 보여?
2026년 1차 청년 매입임대 왜 목록에서 빠졌어?
```

봇은 지원 완료, 관심 없음, 영구 제외 규칙, 마감, 비활성, 중복, 조건 제외, 분석 대기·실패, 실제 다음 페이지 노출을 현재 저장 상태로 구분합니다. 비슷한 공고가 여러 개면 한 메시지에서 번호만 다시 묻습니다. 잘못 누른 `지원했어`나 `관심없어` 때문에 빠진 경우에는 설명 메시지에 `이거 취소해` 또는 `관심없음 취소`라고 답장할 수 있습니다.

## Telegram 핵심 메뉴

- `/help`: 이 사용법의 한 화면 요약
- `/briefing`: 오늘 핵심 브리핑
- `/inbox`: 저장한 일정·할 일
- `/reminders`: 예정 알림 확인·취소
- `/housing_status`: 주택 지원 진행
- `/job_status`: 채용 지원 진행
- `/ask`: 서버 상태·Codex 사용량 질문

고급 명령은 계속 동작하지만 메뉴에서는 숨깁니다. `/help 자세히`에서 전체 명령을 확인할 수 있습니다. `/ask`는 서버 운영 질문에만 사용하며 저장·수정·피드백에는 필요하지 않습니다.

## 봇이 이해하지 못했을 때

봇이 대상이나 날짜를 확신하지 못하면 데이터를 바꾸지 않고 필요한 것만 한 번 묻습니다. 해당 공고·목록·Inbox 확인 메시지에 Telegram `답장`으로 다시 보내면 대상을 가장 쉽게 지정할 수 있습니다. 계속 실패하면 `/help`에서 고정 명령을 확인할 수 있습니다.
